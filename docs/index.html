왜 의미 분석이 필요한가요?

Existing Problem

단순 텍스트 매칭 방식은 기사 제목이 조금만 달라도 중복으로 인식하지 못합니다.

"NYT sues OpenAI" vs "OpenAI hit with lawsuit by New York Times"

Our Solution

문장의 의미를 벡터(Vector)로 변환하여 문맥적 유사도를 분석합니다.

  • 제목이 달라도 같은 내용이면 필터링
  • Gemini Embedding (text-embedding-004) 활용
  • 지능형 임계값 기반 정밀 판정

Core Technology

🧬

Gemini Embedding

텍스트를 768차원의 고차원 벡터로 변환하여 언어의 핵심 의미를 수치화합니다.

📐

Cosine Similarity

두 벡터 간의 각도를 계산하여 문장의 의미적 거리를 측정합니다. (기본 임계값: 0.85)

⚡

Batch Processing

수집된 수십 건의 뉴스를 한 번에 임베딩하여 API 호출 효율을 극대화합니다.

How it works

01

News Fetching

RSS 및 뉴스를 통해 최신 소송 관련 기사를 수집합니다.

02

Vectorization

수집된 제목과 기존 리포트의 제목을 Gemini Embedding 모델로 변환합니다.

03

Semantic Check

코사인 유사도 분석을 통해 중복 여부를 판정하고 중복 시 자동으로 제외합니다.

04

Clean Report

핵심 정보만 담긴 깔끔한 리포트를 GitHub Issue와 Slack으로 발행합니다.
