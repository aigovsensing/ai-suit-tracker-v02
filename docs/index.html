AI Litigation Radar v1.4

AI Lawsuit Monitor

AI 모델 학습을 위한 데이터 무단 사용 및 저작권 소송을 실시간으로 추적하고 분석하는 인텔리전트 자동화 도구입니다.

핵심 기능 보기 GitHub 저장소

실시간 모니터링 & 지능형 분석

단순한 수집을 넘어, 법률적 맥락을 파악하고 리스크를 점수화합니다.

CourtListener 연동

미국 연방법원(PACER/RECAP) 데이터를 실시간으로 조회하여 고소장(Complaint) 및 주요 법원 문서를 수집합니다.

뉴스 기반 보강

RSS 피드를 통해 전 세계 AI 소송 뉴스를 트래킹하고, 관련 사건번호(Docket)를 역추적하여 상세 정보를 보강합니다.

Gemini AI 분석

LLM을 활용하여 수집된 데이터를 요약하고, 의미론적 유사도 분석을 통해 제목이 달라도 동일한 뉴스를 걸러내는 지능형 중복 제거를 수행합니다.

리스크 레벨링

820 Copyright 코드 분석 및 본문 키워드 매칭을 통해 '비인가 학습' 여부를 0~100점 사이의 리스크 레벨로 시각화합니다.

비인가 학습 감지 기준

소장의 내용을 분석하여 저작권 및 학습 관련 쟁점의 강도를 수치화합니다.

🔥
80 - 100 Critical High

무단 수집 + 학습 + 상업적 사용

⚠️
60 - 79 High Risk

모델 학습 직접 언급 및 관련 쟁점

🟡
40 - 59 Medium

학습 데이터 관련 법적 쟁점 존재

🟢
0 - 39 Low

간접 연관 또는 정식 계약 사례

시스템 아키텍처

데이터 수집부터 지능형 필터링, 그리고 리포팅까지의 흐름입니다.

1

Data Ingestion

CourtListener API와 RSS Feed를 통해 최신 소송 및 뉴스 정보를 매 시간마다 수집합니다.

2

Intelligent Extraction

PDF/HTML 본문에서 사건번호, 당사자, 소송 이유, AI 학습 관련 Snippet을 정밀 추출합니다.

3

Semantic Deduplication

Gemini Embedding을 사용하여 이전 리포트와 의미적으로 겹치는 내용을 완벽하게 제거합니다.

4

Multi-Channel Reporting

GitHub Issues에 상세 테이블을 생성하고, Slack을 통해 실시간 요약 알림을 발송합니다.

Technology Stack

Python 3.9+
Google Gemini 1.5
GitHub Actions
Slack API
CourtListener API

Quick Start

환경 변수 설정만으로 즉시 운영이 가능합니다.

# 필수 설정 (GitHub Secrets)
SLACK_WEBHOOK_URL = "https://hooks.slack.com/services/..."
GITHUB_TOKEN = "ghp_your_token..."

# 지능형 기능 활성화 (선택)
GEMINI_API_KEY = "your_gemini_api_key"
GEMINI_SEMANTIC_DEDUP = 1  # 의미론적 중복 제거 활성화
GEMINI_AISUIT_TREND_DAYS = 3  # 동향 요약 기간
