User Guide

AI Lawsuit Radar의 설정 및 사용 방법에 대한 상세 가이드입니다.

1. 프로젝트 개요

AI Lawsuit Radar는 인공지능 학습 데이터와 관련된 저작권 소송을 자동으로 탐색하고 분석하는 도구입니다. 전 세계 법원 데이터와 뉴스 피드를 실시간으로 모니터링하여 핵심적인 법적 리스크를 리포트합니다.

주요 데이터 소스:

  • CourtListener (RECAP): 미국 연방법원의 공식 도켓 및 소장(Complaint) 데이터
  • RSS News Feeds: 글로벌 주요 테크 및 법률 뉴스 매체

2. 핵심 인텔리전스 기능

2.1 비인가 학습 소송 감지 레벨 (Risk Scoring)

소장의 내용을 분석하여 0-100점 사이의 리스크 점수를 산출합니다. 주요 가점 요인은 다음과 같습니다:

  • Copyright 직접 언급: +30점
  • Unauthorized/Scraping 명시: +25점
  • Model Training 직접 언급: +20점
  • Commercial Use: +10점

이 점수를 바탕으로 🟢(Low), 🟡(Medium), ⚠️(High), 🔥(Critical) 등급으로 자동 분류됩니다.

2.2 역사적 추이 분석 (Historical Trend)

2021년부터 현재까지의 데이터를 전수 조사하여 AI 관련 소송의 증가 추세를 분석합니다. src/stats.py 모듈을 통해 연도별 소송 건수를 통계화하고 그래프로 시각화합니다.

2.3 전략적 영향 분석 (Strategic Impact)

단순한 소송 요약을 넘어, 수집된 정보가 기업(예: 삼성전자 가우스, 갤럭시 AI)에 미칠 수 있는 잠재적 영향력을 Gemini가 심층 분석합니다. 법적 대응 전략 및 기술적 대비 사항을 포함합니다.

3. 중복 제거 및 최적화

3.1 3단계 중복 제거 파이프라인

이 프로젝트는 3단계 중복 제거 시스템을 운영합니다:

  • 1단계 - Exact Match: 기사 제목/도켓 번호 완전 일치 즉시 제거 (항상 활성)
  • 2단계 - BM25 유사도: BM25_SEMANTIC_DEDUP = 1 설정 시 활성화. 키워드 빈도 기반으로 유사 표현의 기사를 감지합니다. BM25_DEDUP_THRESHOLD(기본 3.0)로 임계값 조정 가능.
  • 3단계 - 의미론적 임베딩: GEMINI_SEMANTIC_DEDUP = 1 설정 시 활성화. Gemini Embedding(gemini-embedding-2) 모델로 제목이 달라도 같은 사건을 다루는 기사를 감지(코사인 유사도 0.85 이상)합니다.

3.2 교차일 중복 제거 (Cross-Day Dedup)

PREVIOUS_ITEM_DEDUP_DAYS 설정을 통해 과거 이슈에 보고된 내용이 새 이슈에 다시 나타나는 것을 방지합니다. 며칠 간의 히스토리를 대조할지 직접 설정할 수 있습니다.

3.3 시각적 동향 리포트 (Visual Reporting)

GEMINI_DAILY_REPORT_IMAGEGEN = 1 설정 시 금일의 핵심 이슈를 지브리 스타일의 이미지로 시각화합니다. Google의 Imagen 3 모델을 사용하여 텍스트 리포트의 가독성을 높입니다.

4. 이메일 리포트 (HTML Email)

ENABLE_EMAIL_SENDER = 1 설정 시 석간뉴스 및 조간뉴스 리포트를 이메일로 자동 발송합니다. Markdown 본문이 HTML로 변환되어 테이블, 코드블록, Alert 박스가 이메일 클라이언트에서 올바르게 렌더링됩니다.

  • SMTP: Gmail SMTP (smtp.gmail.com:587 / STARTTLS)
  • 인증: Gmail 앱 비밀번호 (SMTP_PASS Secret에 등록)
  • 수신자 설정: data/email.json의 receivers 배열에 이메일 주소 추가
  • 중복 방지: 단일 sendmail() 호출로 전체 수신자에게 1회만 발송
# data/email.json 설정 예시
{
  "smtp_host": "smtp.gmail.com",
  "smtp_port": 587,
  "sender": "your-email@gmail.com",
  "receivers": ["recipient@gmail.com"]
}

5. 시작하기 (Quick Start)

GitHub Actions를 통해 서버 없이 무료로 운영할 수 있습니다.

  • Step 1: 본 저장소를 Fork 합니다.
  • Step 2: Settings > Secrets and variables > Actions 메뉴로 이동합니다.
  • Step 3: 아래의 필수 환경 변수를 등록합니다.
[Secrets]
GEMINI_API_KEY: Google AI Studio API 키
COURTLISTENER_TOKEN: CourtListener API 토큰
SMTP_PASS: Gmail 앱 비밀번호 (16자리)
SLACK_WEBHOOK_URL: Slack Incoming Webhook URL
GITHUB_TOKEN: Issue 및 Commit 권한이 있는 토큰

[Variables]
ENABLE_EMAIL_SENDER: 1 (이메일 발송 활성화)
LOOKBACK_DAYS: 3 (수집 기간, 일)
GEMINI_SEMANTIC_DEDUP: 1 (의미론적 중복 제거)
BM25_SEMANTIC_DEDUP: 1 (BM25 키워드 중복 제거)
PREVIOUS_ITEM_DEDUP_DAYS: 3 (교차일 중복 제거 범위)
